It has never been easier to build your own framework

Building a framework like Next.js used to be hard. These days are over: with modern tools, such as Vite, you can build your own Next.js with only hundreds of lines of code.

Don't let a framework dictate your architecture, keep full control, and use any tool and technology you want.

Build internal company frameworks to scale your teams, or delight your users with a bespoke framework, or just keep architectural control.

Build Your Own Framework.

All-included frameworks

While frameworks like Next.js only take care of the frontend, you can build a framework that includes everything: the server (Express.js, Koa, ...), authentication, API (GraphQL, RPC, ...), ORM (Prisma, MikroORM, ...), logging (e.g. Prometheus), error tracking (Sentry, Bugsnag, ...), emailing (e.g. Mailgun + Nodemailer), deployment (AWS, Cloudflare Workers, ...).

All your framework's users have left to do is: define UI components and data models (if your framework has a database).

Build company internal frameworks, to scale your teams

Give senior employees the power of fully owning the architecture while providing junior employees an all-included framework that takes care of everything.

Keep full control and leverage your company's favorite tools without limitations nor compromise.

You can then share (some of) your internal frameworks with the world, fostering open source values within your company while stimulating and attracting engineering talent. For example, Facebook's decision to open source and share React with the world has been a strong force inside Facebook for engineering excellence and has enabled Facebook to hire highly skilled Software Engineers.

Build a framework tailored to your product, to delight your users

Delight your users with a bespoke framework that deeply integrates with your product.

For example, if you are a headless CMS provider (Sanity, Contentful, Tina, ...), then build a CMS on top. Or, if you are a full-stack deployment provider (PlanetScale, Cloudflare D1, Fly, ...), then build a thin full-stack framework to showcase the capabalities of your deployment platform and for the community to fork. Or, if you are an e-commerce platform, provide a framework like Shopify's Hydrogen.

Build a "Framework as a Product".

Getting Started

Vike is just a website (this website) recommending you to build your own framework.

Following tools can be used to build frameworks.

  • Vite - Next Generation Frontend Tooling.

    Vite is a webpack alternative that has first-class support for building frameworks.

    In particular, check out Vite's native SSR API which is a low-level and highly flexible API enabling you to build SSR frameworks.

  • vite-plugin-ssr - Like Next.js/Nuxt but as do-one-thing-do-it-well Vite plugin.

    Vite-plugin-ssr also has first-class support for building framworks. It takes care of significantly more things than Vite's native SSR API: it enables you to build a framework like Next.js with only hundreds of lines of code.

    If you need more flexibility then use Vite's native SSR API instead.

    Check out its guide vite-plugin-ssr > Build Your Own Framework.

  • react-streaming - React 18 Streaming. Full-fledged & Easy.

    Easily integrate React 18's SSR streaming advanced capabilities such as server-side <Suspense>.

  • HatTip - (nothing) Like Express.js

    Easily integrate your framework with any deployment provider (Cloudflare Workers, Vercel, ...).
